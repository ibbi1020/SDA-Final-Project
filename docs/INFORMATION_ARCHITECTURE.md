# Block20 Gym Management System - Information Architecture

## Document Overview
This document defines the information architecture (IA) for Block20's dual-portal system, designed around modern UX principles: **discoverability**, **simplicity**, **task-orientation**, and **minimal navigation**. The system provides role-based access with tailored experiences for Staff and Members. This is a Desktop Only App, no mobile intefaces.

**Version:** 1.0  
**Last Updated:** November 2025  
**Design Approach:** Progressive Disclosure, Information Scent, Task-Driven Navigation

---

## Design Principles

### 1. Task-Driven Organization
- Group navigation by user goals, not system functions
- Most frequent tasks appear at top level
- Minimize depth: 2-3 levels maximum for any task

### 2. Progressive Disclosure
- Show only relevant information for current context
- Hide advanced options until needed
- Reduce cognitive load on initial view

### 3. Consistent Navigation
- Primary navigation persistent across portals
- Familiar patterns and terminology
- Clear wayfinding indicators (breadcrumbs, current page highlight)

### 5. Minimal Clicks to Goal
- Primary actions 1-2 clicks from dashboard
- Frequently used functions always visible
- Search/filter for longer lists

| Feature | Staff Portal | Member Portal | Notes |

## Navigation Principles Summary

### 1. **Information Scent**
- Users can predict content from navigation labels
- No confusing abbreviations or jargon
- Clear path from need to solution

### 2. **Progressive Disclosure**
- Overview screens show most critical info
- Details available on click/expand
- Advanced options hidden but accessible
- Example: Membership status â†’ click for full details â†’ edit option

### 3. **Task Grouping**
- Group by user goal, not system function
- Related actions co-located
- Reduces search time
- Example: "Renew Membership" near "Membership Details" not under "Billing"

### 4. **Consistent Patterns**
- Same navigation style across portals (adapted for role)
- Consistent button placement and labeling
- Predictable workflows
- Users build mental models faster

### 5. **Accessibility & Mobile**
- Touch targets: minimum 48px Ã— 48px
- Readable on small screens
- Clear focus indicators
- WCAG 2.1 AA compliance
- Keyboard navigation support
