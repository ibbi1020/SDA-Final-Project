## Use Case 1: Manage Member Profiles

| Element | Description |
|---------|-------------|

| **UC Scope** | Block20 Gym Management System |

| **Post-conditions** | - Member profile is created/updated in the database<br>- Unique member ID is assigned (for new members)<br>- All data validations are passed<br>- Audit trail is created for the transaction |
| **Main Success Scenario** | 1. Administrator selects "Manage Members" option<br>2. System displays member management interface<br>3. System displays confirmation message<br>9. System generates member card (for new members) |
| **Alternative Success Scenario** | **3a. Deactivate Member Account:**<br>1. Administrator selects member to deactivate<br>2. System prompts for deactivation reason<br>3. System checks for outstanding dues<br>4. System updates member status to inactive<br>5. System disables access credentials<br><br>**5a. Duplicate Member Found:**<br>1. System displays potential duplicate records<br>2. Administrator verifies if genuine duplicate or different person<br>3. Administrator either merges records or continues with new entry<br><br>**5b. | **Occurrence Frequency** | Approx. 5 times per day |
